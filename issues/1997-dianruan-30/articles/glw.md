# CD 音源的劲敌——G.L.W

文／责编／PsychOIC

有没有过这样的经历：从好友处借来了一张很好听的 CD，还没有听够，就被催着要回；想到录成 .wav 的文件以后慢慢听，但是发现如果用 Full Quality（注 1）去录制，每一分钟要占用 10 兆的空间，如果采用压缩模式，也省不了多少空间，而且 20Hz 以下和 20KHz 以上的数据会被抹去，如果再考虑降低质量，空间到是省了一些，可是录出来的东西越听越不对味儿，怎么办？就这样放弃了吗？

一般方法是无法解决的，不过 G.L.W 可以解决。G.L.W 是什么？这是三个软件，用这三个软件，就可以制作出高品质低容量的音乐文件了。G 是 Grabber 软件，Grab 是抓取，那么 G 软件就是 CD 音轨抓取软件。L 是压缩软件，全名 Layer 3 encode & decode，即第三层加解码，用来压缩大容量的音频文件。W 是 Winplay3 的缩写，也就是播放软件，因为用 L 压缩出来的文件，一般播放软件是不能放的，目前只有 WinPlay3 可以播放。

如果对 G.L.W 的能力表示怀疑，请看下面的试验报告。笔者用 Speed 的《Body & Soul》做测试。这首歌长 5 分 2 秒，用 Full Quality 抓取音轨，然后还采用 Full Quality（使用 128000bitrate）压缩，产生的文件大小如下：

BDY & SOUL.WAV　　53,415,740 bytes
BDY & SOUL.MP3　　4,845,818 bytes

用 Winplay3 播放 bdy & soul.mp3，由于完全采用 Full Quality，所以几乎听不出来不同（实际上差异还是有的，只是人耳听不出来，或是说一般人耳听不出来，如果您是音响器材发烧友，那本人无话可说）。设想一下，一首 5 分钟的 CD 音质的采样只用 5 兆空间，平均一分钟占用 1 兆空间，那么一张 CD 按 650 兆算的话，可以存放 650 分钟，共 130 首 5 分钟长的歌，远比一张音乐 CD 的十几首歌要多得多。怎么样，跃跃欲试了吗？不要着急，听本人来介绍一下这几个软件。

## Grabber

抓音轨的软件其实现在已经有很多种了，笔者常用的是 Digital Audio Copy 2.2，简称 DAC，其他还有一些如 CDDA，DA2WAV 等软件，但是不太好用（本人个人观点）。DAC 是一个 CD 播放与抓取二合一的软件，不过播放功能不是我们所看中的，因为 DAC 的播放功能实在是普通到了没什么可讲的。在 DAC 的抓取功能设置里，我们可以根据自己的需要设置采样频率（11KHz，22KHz，44KHz），采样比特（8bit，16bit）和采样效果（单声道，立体声，左声道，右声道）。效果越好，所需要的硬盘容量就越大，相反就越来越小。如一首长 3 分 47.97 秒的歌，一共 17098 sectors，抓取时间为 5 分 30 秒，用 44KHz，16bit 立体声采样抓出来的采样文件为 40,072,872 bytes，而一首歌用 11KHz，8bit 单声道采样生成的文件只有 2,504,755 bytes。基本抓取时间与采样方式没有太大关系，只取决于设备的速度。不过 DAC 有一些缺点。一、要使用 DAC，光驱必需有可以将音轨当作数据读取的功能。这一点使得不少人不能使用 DAC。笔者使用的松下四速与八速光驱，都可以使用 DAC。然而老 TEAC 四倍速，米苏米系列可能就不太走运了（其他品牌 p 牌因没有条件试验，不敢妄下定论）。二、在 WIN95 中，每张 CD 的第一轨都有可能不能抓取。这个问题估计出在 WIN95 的光驱驱动程序上，在 DOS 中则没有出现这个问题。三、经常会有 bug，如开始抓取音轨后只有光驱在转动，而硬盘不动，进度尺也不动。这时只要强行按光驱开门键打开光驱，然后再放入重新工作即可。DAC 是一个 30 天试用的共享软件，另外还有 TrackTime 和 Edit CD Time 两项功能只有在正式版中才可以使用。

## Layer 3

可以说 G.L.W 的精髓就在 Layer 3 加解码软件了，这可是目前世界上唯一的一个 Layer 3 的加解码软件。Layer 3 分为 l3enc.exe（加码）与 l3dec.exe（解码）两部分，不过常用的只是 l3enc.exe。能把一首录完后要 50 兆的歌压缩到 5 兆，您一定觉得不可思义，其实这没有什么新鲜的，想想看一部 2 个小时的电影，包括了大量的声音与影像数据，而用 MPEG 标准压缩后也就两张 CD。Layer 3 就是使用 MPEG 标准技术去压缩音频信号。Layer 3 的音频压缩技术从 94 年时就开始开发了，但是直到今年才基本完善成形。其实 Layer 3 是音频数据压缩的第三级，除 Layer3 外还有 Layer 1 与 Layer 2。Layer 1（压缩比为 1：4）Layer 2（压缩比为 1：6－1：8）Layer 3（压缩比为 1：10－1：12）。

l3enc 允许用户自行设定压缩的一些参数，如输出比特率 bit/s（范围从 8000br 到 256000br，缺省为 112000br，不过目前“国际”流行的大多是 128000br），输出质量（加 -hq 参数为高质量），输出效果（加 -dm 为 downmix 注 2）等等。输入的文件不只限于 .wav 文件，还有 .aif，.snd，.au 等等。

如果您没有用过 l3enc，您可能想象不到压缩一首歌是多么的漫长。笔者用编辑部的 486/100，16 兆内存，win95 下压缩那首《Body & Soul》用了 140 分钟。不过据说在 Pentium 133 上压缩一首 5 分钟的歌只用 50 分钟到一个小时。关于使用环境，人说在 DOS 中加码速度要比在 WIN95 的 DOS PROMPT 下快一些，经过本人的反复试验，还是建议在 WIN95 下使用比较好一点，因为尽管多工有可能会拖慢速度，但是至少还可以同时运行其他的软件，而在 DOS 中就只能看着屏幕干等了。Layer 3 encode & decode 也是一个 30 天试用的共享软件。

## Winplay3

用 l3enc 压缩后的文件是以 .mp3 为后缀的，而且又是压缩格式，所以必需一个特别的播放工具，就是 Winplay3。播放 MPEG 音频压缩文件如同播放 MPEG VCD 一样，不同档次的 CPU 的回放效果也不一样，下面是 Winplay3 在不同的 CPU 上回放性能的比较：（Intel 系列 CPU）

|  | Pentium | 486/66 | 486/50 | 486/33 |
|---|---|---|---|---|
| MPEG－1 stereo | yes | no | no | no |
| MPEG－1 downmix | yes | yes | no | no |
| MPEG－1 mono | yes | yes | yes | no |
| MPEG－2 stereo | yes | yes | yes | no |
| MPEG－2 downmix | yes | yes | yes | yes[?] |
| MPEG－2 mono[?] | yes | yes | yes | yes[?] |

在笔者用的 486/100，16 兆内存，系统为 Win95 的机器上，可以很流畅的以 22Khz，16bit，立体声方式回放歌曲。但是如果用 44KHz 回放，或是 win95 运行了多个任务（哪怕只多运行一个任务），在回放几十秒钟后都会“跳格”。不过这种状况在 Pentium 上是不会出现的。Winplay3 中还设有 Play Location 的功能，也就是说支持在 Internet 上 Online 播放 .mp3，只要在 URL 栏中添入正确的地址，就可以在 Internet 上边“冲浪”边的“音乐”了（可惜国内 ISP 的速度太慢了，否则真是要试一试）。Winplay3 是一个 20 秒钟回放试用的共享软件（每次只放 20 秒就会停）。

## 后话

以 G.L.W 目前的水准，如果用来制作音乐 CD 的话，实在是不适合的。首先用这种方法制作出来的 mp3，一般玩家听着可能觉得不错，但以专业标准来看，比真正的 CD 音源还是有很大差距的。其次，用来回放的硬件标准也不低（如果是以 Full Quality 回放），而一般音乐 CD 只要有光驱就可以播放了。第三，制作时间太长了。曾经遇到过一位美国的 MP3 玩友，玩 MP3 疯狂到了编写一个“抓第一音轨→加码→删第一轨采样→抓第二轨→加码……”的批处理，在一台 P133 上夜以继日的不停地制作 MP3，真是中毒不浅了。所以，一般玩家对于这项技术，只能是“拿来玩玩”，自己作一两个 .mp3 试试，或者是把借来的音乐 CD 抓一两轨留下来。而对于多媒体制作工作者来说，到是可以好好利用一下，制作一些高品质的音乐音效。

注解：
1、Full Quality：完全高质量，采样或回放时采用 44KHz，16bit，立体声。
2、downmix：将立体声数据输入，以单声道方式回放。
