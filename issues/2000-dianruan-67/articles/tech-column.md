### 新主机招牌处理特色解析

西南师大计算机系 赵欣

天语：科普园地兼顾实用性和发烧性，硬件原理方面的文章同样欢迎，望专业人士不吝赐数

SE、N、SO不遗余力地将各自主机的3D功能强化，不过由于各种各样的原因，三家在硬件上的差别很大。但是，作为几个集团（每部主机的娘家都有好几个，而且均为超级大公司）精心推出的产品，3种主机在3D处理方面都有各自的特点，以此来满足厂商的低价格（游戏机的基本特征）高性能的要求。而且其中的某些技术到目前为止甚至将来的一段时间内还是独一无二的，有较大的前瞻性。这里，将已公开的资料作进一步的分析，希望读者增加这方面的知识的同时还能够对新一代游戏机有更进一步的认识。

首先想说一下游戏机厂商很喜欢拿主机的多边形（POLYGON）生成能力来炫耀自己的主机。实际上多边形处理速度只是限制3D游戏画面场景的复杂程度，而另外一个重要指标则是决定3D游戏帧速度（即流畅度）的关键：像素填充率！这两个指标哪一项先达到瓶颈，哪一个就成为3D处理中的拖累，不过由于目前游戏机仍然绝大多数使用TV作为监示器，而TV的分辨率有很大限制，使得游戏机的3D部分对填充率的要求不像在PC上那么高，不过由于未来的各种3D特效有的十分耗费填充率，所以为了保持更好的帧速，高填充率是必要的，正比这块的内存的数量一样，不嫌多啊！

一般来说传统的3D加速芯片（包括PS2的GS芯片。此处 + r1 + 是指处理方法，下文会有解释）的填充率计算公式为：时钟频率×纹理单元的个数。例：150MHz的芯片有2个纹理单元，那么理论值填充率为150×2＝300M pixels/sec或300M texels/sec。不过要注意，以上数值只是理论上3D芯片100%的时钟周期都来做这个工作所得到的。但这是不可能的。实际上的填充率比这个值要低些（Z-BUFFER&帧缓存清除、翻页时的同步以及处理3D图象时巨大的数据量）。不过，DC在这方面有着自己的独到之处，其原因就是DC使用了NEC&Video Logic出品的POWER VR系列芯片。这款芯片的设计是与众不同的。

**DC、海豚：等效填充和纹理压缩**

传统加速芯片产生的一个问题是由于许多多边形组成，在渲染时全部都渲染——不管我们看不看得见——这就造成了像素的极大浪费，因为有些像素渲染出来根本就没有用！而DC就不会有这种浪费。DC的芯片就无法直接按比较填充率指标和别的芯片。因为所用技术根本不同，于是Video Logic提出了一个等效填充率的概念，以便其和别人的3D加速芯片有了可比性。

例：一面墙和一个有一半墙那么大的怪物，传统的3D卡会把整面墙和怪物都渲染出来，也就是说它了1.5个屏幕，而DC却不会将被怪物挡住的那半截墙面画出来，也就是只画了1个屏幕——而实际上我们也只能看到这一个屏幕上的东西。这样就节约了另外半面墙的像素资源。等效填充率的公式也就出来了：

（被挡住部分像素所占屏幕百分比（Overdraw，上例中为50%）+1）×理论填充率。

上例中若理论值为100M pixels/s的话，那么等效值为（1+0.5）×100＝150M pixels/s。一般的游戏中Overdraw的值在0.2～3之间，换句话说传统的3D芯片额外多画了0.2～3个屏幕，这对提高帧速极为不利。而DC所使用的技术使其在较低的主频下就可以拥有较高的性能。其实在科技日新月异的今天，单纯用 + r1 + 、 + r1 + 等粗线条指标来衡量一台游戏机未必太含糊。

面临到了TV分辨率的问题。这是一件事也许常常因为电视的原因被TV GAME开发者名忽视一般贴图效果。纹理贴图就是在多边形骨架上贴上图案（2D），使3D图象成型。由于分辨率过低，当我们把视角拉近的时候不会表现真实情形下那样越来越清楚，反而越来越 + r1 + 。不过如果我们使用大纹理的话（也就是分辨率和色深度量和色深高点啦），情况就不同了，物体拉近的时会变得更有层次感和色深高点啦）。

不过光一个2048×2048×32bit的贴图就有16MB大（MY GOD！！）。这种海量显存谁敢乱用！？

还好有纹理压缩技术——S3TC（For海豚）以及VQTC（For DC）。

海豚上使用的S3TC（S3 Texture Compression）是S3公司的专利技术，它通过一套专用结构和算法来处分纹理管线之中的数据，在进行渲染之前就行压缩，压缩比为6:1。这使得游戏中大纹理的使用变得很有希望（不过不会有2048×2048那么夸张）。除了海豚，DC也拥有自己的压缩技术——VQTC（Vector Quantization Texture Compression），即矢量化纹理压缩。这也是POWER VR芯片所提供的独有的（投机！？）技术。VQTC对纹理作分块处理，合成很多2×2的小栅格，再进行比较，如果它认为一些小块的样子差不多（1？），那它就把它们算作一样并只记录这一个子的信息，和那些不丢弃的做法信息约的资源——压缩比与S3TC相比更高几倍，达到17:1。信不信由你。而且这种有损压缩出来的质量表现也不算差！不过当然比不过S3TC的了。

### PS2：4MB显存和D-RD RAM

说到SONY的PS2，首先让人想到的就是它的那一大堆 + r1 + 的技术指标。先不说PS2是否真有能力达到其所标称的性能，但是PS2所使用的一项技术确实对3D性能的提高是非常显著的，那就是其GS芯片中的4MB Embedded Cache（嵌入式内存）。

在3D加速芯片中，最制约3D性能提高的不是内存的速度，而是显示系统的带宽！带宽比马路，外面的数据进不来，处理好的东西出不去，则再好的芯片性能也是白费。因此就像CPU一样，显示芯片内部也设计了复杂的多级级存结构来配合3D流水线工作，高的时钟频率和接口位宽数能明显提高带宽。不过要这样做不容易，因为代价太大了。然而，在芯片内部集成Cache就不同了，毕竟再复杂也不过是在芯片内部多几层电路连线，成本也没不容忍。

由于现在的显存容量已经高达数十MB，要缓冲这么大的低速显存，Cache的容量还不能太小，至少上MB（PS2就有4MB）。由于硅片面积越大越贵（半导体的特征），因此采用0.18微米技术生产是必要的，索尼也是这样做的。不然芯片内部如此复杂要上到较高的时钟频率很困难的，而且通过公式，加大接口宽度也是增大带宽的办法之一。这样能使同时工作的通道数目加倍，因此PS2的2560位的接口宽度诞生了（不过这是很夸张）。PS2的显示器部分带宽达到150MHz×2560/8＝48GB/s。真是太可怕用！（尽管如此，KONAMI的大井康建在接受美国PS专门志采访时的仍然认为PS2 + r1 + ）

另外PS2还使用了高速的Direct RDRAM，比普通的SDRAM或SGRAM都快很多，可以说PS2的带宽完全不尤其，只要不乱来，应付各种复杂多变的情况都是绰绰有余的。
