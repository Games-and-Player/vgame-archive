## 来自游戏制作的现场（二）

（PDF p.35 / 刊页 43）

这一次，我就以我社的作品为例，说明一下 SLG（模拟游戏）开发的过程，这时候游戏机上可发布的游戏可说是数量惊人，已经超越了在 Mac 版的一个策略 SLG（游戏攻略）一套 SLG 游戏中，它是一套为 SLG 的专门 SLG 游戏，为我们的最强 SLG 游戏奠定了基础，在这里将开发一款名称为“……”（第一回）。

### 1. 题目、题材

先从题目说起，要先确定你想要开发什么主题？望想对读者来说，怎样才能给读者带来刺激？

基本上，（盖伊弗莱姆[?]）这已经别别不属于外界特殊行动者之外了，因为对外界来说它已经是第二个版本的策略 SLG，并且已经是一个唯一 SLG 的最知名版本，这一个概念可以说是最好的，也是一套最知名的 SLG，不只是 SLG 外观而已，在这里就用“战场”（大意如此）用一个新版本游戏出版了——一回先发布一次……

### 2. 立足于开发之时

有时候游戏公司会担心产品的稳定性，会担心公司在这个世代的市场竞争力问题，有很多时候，这时候 CPU 的能力会，即这时候 CPU 的能力，为了开发的时候，采取一个模拟实战的全程策略，把游戏里的各种数据参数化，把整个全程游戏画面的动画与游戏系统、玩家界面、游戏体验与目标等一一计划与整理数据库，进行后……

2. 其次，运行 CPU 战斗系统。一定要开发 CPU 战斗机的策略运行，而由于 CPU 的能力，为了能对各方面比如 CPU 的 K、M、M 值进行区分开来，必须要首先先了解清楚设定的不同参数和机器性能的不同等级，从单个到整体到 CPU 的 K、M 值的变化——

3. 在处理方面：CPU 的游戏战斗策略对整个处理流程、对战双方各自的影响，一般是从最大值的设定，进行全部的数据设置和运算的，正常情况来说，游戏机在 CPU 的控制力成功地在某方面形成了一个策略集合体，进行一个全面的整体控制。

---

（PDF p.36 / 刊页 44）

这次时候有些事务必要注意，进行了大量测试调整之后，得出一下那些测试结果，况且，一定要经常进行测试调整，所以，玩家来说，最终目的永远是在游戏中完成了任务，获得满足感，所以测试结果必须能满足玩家最终游戏目标，在这个方向是（Y），所以测试的时候应当注意到 CPU 战斗难度的精细调整，以令游戏在战斗时合乎“玩家对战斗结果的合理预期”，让玩家觉得 CPU 的这种“运作方式”是“合情合理”的，而非“作弊”的。

所有的人如果切换对对方形式来看，这也就是（CPU）战争策略最终的游戏目标，做到最低最低标准就是让游戏者最终感到满足、得到乐趣，这就是一个良好的游戏设计应当追求的最基本的出发点。
