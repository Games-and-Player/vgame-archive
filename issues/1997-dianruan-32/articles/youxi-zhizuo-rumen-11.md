## 从分辨率谈起：TOUCH 的坐标系

视频游戏对于人物角色的碰撞检测来说，关键在于处理 sprite 之间、人物与地面等各类碰撞情形。

先分辨率说起比较好。如果要处理各个对象（sprite）之间的 TOUCH，只考虑位置是不行的，必须加上尺寸。小的 sprite，为了节省运算量，常常用一个点代替整个图块来处理碰撞，但较大的 sprite，特别是大量的 sprite，如格斗游戏中的人物碰撞，就不能不做多矩形精细处理了。

设分辨率为 320×240，左上角坐标为（0,0），右下角为（319,239）。

以格斗游戏来讲，两个角色以 RYU 和 KEN 为例说明 TOUCH 处理方法：

```
        RYU: A (x, y)          KEN: B (a, b)
站立    {a, y}                {(x+a, y+b)}
蹲伏    {(x, y) + (x+a, b)}  {(x, y) - (x+a, y+b)}
跳跃    {a, y}                {(x+a, y+b)}
```

```
RYU:{x 全屏标中坐标(x,y)}=RYU{(x,y)}
RYU→ x(y)→(x+a,y+b)……
RYU→ x(y)→(x+a,y+b)……
RYU：全屏标中坐标(x,y)→(a+x,y+b)……
```

```
RYU：y 为最顶的坐标(x,y)，播……
RYU→ x(y)→(x+a,y+b)……

一加前交替打，再判断各角色格。
一顺序、滑打。
```

## 从分辨率谈起

先说 TOUCH 处理实际的数量，首先是最常用到的矩形 TOUCH 分析……

假设两个人物 RYU 和 KEN 在格斗游戏中，要分析他们的 TOUCH 碰撞体积，要描述 RYU 的角色坐标（x,y）和 KEN 坐标（a,b），以及各自的判断矩形宽高（w,h），则碰撞判断可以用以下公式检测：

```
RYU:{全屏标中 (x,y)} = RYU{(x,y)}
RYU→ x(s)→(x,y)→(a+x,y+b)……
RYU→ x(s)→(x,y)→(a+x,y+b)……
RYU：全屏标中坐标(x,y)→(a+x,y+b)……
```

总之，格斗游戏中的角色 TOUCH 判断需要多组矩形叠加，不同招式对应不同的判定框大小。这是 ACT 和 FGT 等游戏类型共通的核心技术。

一顺序打好了，就检查到了所有角色的 sprite 上面，好了！

## 角色分量碰撞处理

格斗游戏碰撞判断的关键，是对 STREET FIGHTER 系等游戏的角色判断框体系的了解。

以拳皇系列的角色碰撞分析来说，角色主体被拆解为若干判断矩形：

- 身体矩形（全体接触判定）
- 攻击矩形（攻击判定）
- 受击矩形（受击判定）

每个矩形用左上角和右下角坐标来描述，处理逻辑如下：

如果 RYU{x,(y,b)} 的坐标矩形（不在实际坐标内 sprite 上的 RYU）的 sprite 的坐标 y 到 KEN 的 x 的所有矩形方向上方向，那这个 sprite 位于两个最左侧 RYU{(x,y)} 的坐标，以 KEN 的对比 A(x,y), B(x,y) 来看，两个角色的矩形会有左标 + 右标的重合区，从而触发碰撞事件。

```
RYU：y 为最顶的坐标(x,y)，（坐标范围记）……
RYU→ x(y)→(x,y)→(a+x,y+b)=0

一加前交替打，再判断各角色格。
一顺序、滑打。
```

如果是 STREET FIGHTER 类 ACT 格斗游戏，每一个 sprite 的 FGT 从纵向 ACT 来处理，这些游戏的碰撞就是以 sprite 的 FGT 矩形 + sprite 的 ACT 的整体来描述了。
