# 游戏制作入门（五）

## SPRIT与卷轴

噼噼啪啪——魔法师再次登场！各路玩儿，上回讲的是了解吧，某某上回的某某知识，GO ON！

SPRIT，就是前景的移动块，往几大页面上运行的数量较多。玩家控制的大字符空格移动，这个很大的定点力方形板，就是 SPRIT，大内行么 GAME 地，SPRIT 是游戏机所特有的，目的是使再多色彩移动能力，因此允许之不必定义很多内存格而来回，这样对 SPRIT 的大小有限制，不能不超过一定量少呢，PC 上就没有这些打，可以自定义最新数以实现多 SPRIT 的设置。

GET 和 PUT 类似的系统，大多数游戏用 GET 和 PUT 来处理 SPRIT，用的就是一个完整游戏，游戏理由是带有更多的类型。在一个完整的游戏中，SPRIT 是将储存有特殊形图数出来，这种不用来控制 GAME 中进行人物上下左右运动的数组而下的几块。而当 GAME 中进行角色切换时候，整个游戏的画面就通过的方式，SPRIT 切换来表示。

游戏中背景可分 2 种，不同移动的固定背景和可移动的动卷背景。当定好背景和 GAME，如（BOMBER MAN），我们就都大致清楚了。

如果做好背景之后，各人需要一些大的特别的背景，主角在其中移动，可以按照背景特色自由设定，这样的图形可以像上面这样，游戏中人物在上面跑动行走，这种是带有自由移动能力的卷轴背景（卷轴）。

当人物走到背景边界时，如有背景则它不能向外于背景区域，这时人物的行走方式就完了，所以固定对人物行走的固定背景上，固定了移动量才多的。

还要讲一点，关于背景的构造，把背景背景作为一幅地图处理，看着把背景作为一幅地图处理。

---

### 1. 纵轴键：纵景上下移动，如游戏机的（卷轴）。

### 2. 横幅键：分背景左右移动，如从左横（卷轴）。

### 3. 自动键：让上下左右自动移动，如在 NPG 来说还过的话了。

### 4. 背景点：从左下角开始计来背景，如从 SPC 上的（ZERO）。

### 5. 伪 3D 背景：利用层叠卷轴图像密叠，如 SPC 上的（ZERO）。

### 6．3D 背景：VR 向度的真实空间内的移动，RPG 上的（大成界移动），而不移动移动能力大型分的。

SPRIT 在卷轴上的移动：

背景是由图层多形构成的一幅画面，是图层多形的坐标用下来，人们来行走时出现画面 1，2，3…N 层图，于是就在一个数的画面的图，都是一样的图片大下，各个图层高同分辨率，但图层上颜色的深度可以不同，来形成不同的画面效果，还可以对各图层进行显示／隐藏切换。

当上层显示背景的一部分，如图，当玩家是把人物上下左右移动时，而上层是底下的人物画面，不由中在人物背景上行走移动，而是以严格人物在背景上行走移动感，因此只需要用 GET 和 PUT 这关于，这画面里前景健理解一下帮控管理，调理显示上层的图表关于背景数组 K 的是的那它（卷轴之类地）。

当人物走到背景边界时，如有背景则它不能向外于背景区域，当超过边界时，人物的移动方式就完了，所以固定对人物行走的固定背景上固定了移动多的。
